# PJ-AG4：LLM 驱动的博弈/市场模拟 Agent

## 项目设计与实现方案

## 场景版本：AI 算力芯片高端 GPU 现货与供应链博弈

## 一、项目背景与研究目标

本项目构建一个由多个 LLM Agent 参与的动态市场博弈系统，用于模拟高端 AI 算力节点现货与供应链分销市场中的重复竞争与协作行为。市场中的参与者为面向科技巨头、AI 实验室和大模型公司提供高端 GPU 集群节点的全球算力供应商与芯片分销商。交易对象可抽象为标准化的高端 AI 算力商品，例如：

$$
q_t \equiv \text{8卡 H200 服务器节点数}
$$

该场景中的库存具有明显的技术资产属性。GPU 节点若未及时出租或转售，会持续产生机房托管、电力和资金占用成本，同时在新一代芯片上市后面临较快贬值。因此，库存损耗统一建模为：

- 技术折旧与资产闲置损失，Tech Obsolescence
- SLA 违约金与客户流失
- 算力联邦与集群拆借中的同行关系约束

本项目的目标是让 3 个 LLM Agent 在动态需求、有限理性和声誉约束下，对高端 GPU 节点进行定价、备货、履约与同行拆借决策，并研究：

1. 不同经营风格 Agent 的演化收益差异；
2. 时间序列需求波动如何影响 LLM Agent 的预测与决策质量；
3. 声誉机制如何约束价格战、违约与协作行为；
4. 有限理性和信息不对称下，市场是否形成稳定格局、周期性价格战或寡头分化。

## 1. 核心场景与博弈机制设定

### 1.1 市场场景定义

市场中存在 3 个独立决策的 Agent，它们均扮演全球高端 GPU 算力节点现货供应商或二级云分销商。每一轮 $t$ 可解释为一个固定交易周期，例如 1 天或 1 个结算时段。每个 Agent 同时决定：

- 对外报价 $p_{i,t}$：每单位标准算力节点的租赁或转售价格；
- 本轮可调度新增节点量 $q_{i,t}$：可理解为采购、锁仓、预留或调入的新增可交付节点数。

市场总需求 $D_t$ 由科技公司和 AI 实验室的训练与推理集群需求构成，并在 3 家供应商之间按价格、品牌与声誉进行分配。

若某家供应商无法按承诺交付，则会产生：

- SLA 违约罚款；
- 大客户流失风险；
- 声誉下降；
- 后续同行不愿向其紧急拆借算力。

若备货过多，则会产生：

- IDC 机房能耗成本；
- GPU 闲置占用成本；
- 技术折旧损失；
- 现金流压力。

### 1.2 三个 Agent 的角色定位与决策目标

#### Agent 1：Hyperscaler / 规模主导型

- 资金雄厚，能大规模锁定和吞吐 GPU 硬件；
- 倾向用激进价格争夺大客户订单；
- 更愿意承受短期利润波动，换取长期市场份额；
- 风险：过度价格战会损害同行关系，并在需求错判时形成大量闲置算力。

#### Agent 2：Premium Cloud / 稳健交付型

- 品牌溢价高，强调高可用性、数据安全与 SLA 稳定交付；
- 定价相对坚挺，不轻易降价；
- 追求较低违约率与较高声誉；
- 风险：定价过高可能在需求弱周期中失去市场份额。

#### Agent 3：Spot Broker / 现货套利型

- 轻资产运行，依赖现货市场低买高卖与灵活调价；
- 对市场波动响应快，频繁调整报价和持仓；
- 追求高周转和短期套利；
- 风险：在大客户稳定性和长期信誉上处于劣势。

### 1.3 状态变量定义

对每个 Agent $i \in \{1,2,3\}$，每轮 $t$ 定义：

- $p_{i,t}$：Agent $i$ 当轮对外报价
- $q_{i,t}$：Agent $i$ 当轮新增可调度节点量
- $I_{i,t}$：轮初已持有但尚未出租或售出的可用节点库存
- $A_{i,t}$：轮内总可售供给
- $R_{i,t}$：轮初声誉值
- $x_{i,t}$：最终成交或交付节点量
- $L_{i,t}$：轮末剩余未成交节点量
- $short_{i,t}$：缺货或违约数量
- $\pi_{i,t}$：当轮利润

其中总可售供给定义为：

$$
A_{i,t} = I_{i,t} + q_{i,t}
$$

### 1.4 需求分配机制

市场总需求 $D_t$ 由价格、品牌和声誉共同决定分配。定义 Agent 的市场吸引力：

$$
a_{i,t} = b_i + \beta_R R_{i,t} - \beta_p p_{i,t}
$$

其中：

- $b_i$：品牌基础吸引力
- $\beta_R$：声誉对需求吸引力的权重
- $\beta_p$：价格对需求吸引力的敏感系数

再定义 logit 型份额函数：

$$
s_{i,t} = \frac{\exp(a_{i,t})}{\sum_{j=1}^{3} \exp(a_{j,t})}
$$

则分配到 Agent $i$ 的潜在需求量为：

$$
y_{i,t} = D_t \cdot s_{i,t}
$$

这意味着：

- 报价越高，订单份额通常越低；
- 声誉高和品牌强可以支撑更高定价；
- 市场份额连续变化，便于后续做敏感性分析。

### 1.5 初始缺口与富余供给

在同行拆借发生前，Agent 的初步缺口与剩余为：

$$
short^{(0)}_{i,t} = \max(0,\ y_{i,t} - A_{i,t})
$$

$$
surplus^{(0)}_{i,t} = \max(0,\ A_{i,t} - y_{i,t})
$$

### 1.6 同行拆借机制：算力联邦与集群算力拆借

当某家供应商无法满足自己的客户订单时，可以向其他供应商高价紧急拆借算力节点。这一过程模拟现实中的：

- 联邦云临时调度；
- 二级市场 GPU 租赁；
- 同行机房跨账户转租；
- 私下算力票据调配。

#### 合作接受概率

若 Agent $i$ 缺货，而 Agent $j$ 有富余，则 $j$ 是否愿意对 $i$ 提供节点，由下式决定：

$$
P^{\text{coop}}_{j \to i,t}
=
\sigma \left(
\alpha_0 + \alpha_1 R_{i,t}
- \alpha_2 \frac{y_{j,t}}{A_{j,t} + 1}
- \alpha_3 dump_{i,t-1}
\right)
$$

其中：

$$
\sigma(x) = \frac{1}{1 + e^{-x}}
$$

解释如下：

- $R_{i,t}$ 高：说明 $i$ 平时信誉较好，同行更愿意救急；
- $\frac{y_{j,t}}{A_{j,t}+1}$ 高：说明 $j$ 自己也很忙，余量有限；
- $dump_{i,t-1}=1$：说明 $i$ 上一轮恶意发动价格战，同行会降低合作意愿。

#### 实际拆借量

$$
m_{j \to i,t}
=
z_{j \to i,t} \cdot
\min \left(
surplus^{rem}_{j,t},
short^{rem}_{i,t},
\bar m
\right)
$$

$$
z_{j \to i,t} \sim Bernoulli(P^{\text{coop}}_{j \to i,t})
$$

其中 $\bar m$ 为单笔最大拆借量。

#### 拆借后的有效供给

$$
\tilde A_{i,t}
=
A_{i,t}
+ \sum_{j \neq i} m_{j \to i,t}
- \sum_{k \neq i} m_{i \to k,t}
$$

### 1.7 成交量、违约量与技术折旧

#### 最终成交量

$$
x_{i,t} = \min(\tilde A_{i,t}, y_{i,t})
$$

#### 最终违约或缺货量

$$
short_{i,t} = y_{i,t} - x_{i,t}
$$

#### 轮末未利用算力节点

$$
L_{i,t} = \tilde A_{i,t} - x_{i,t}
$$

#### 技术折旧与闲置损失

传统模型中的腐损率 $\delta_i$ 在本项目中定义为技术折旧率与资产闲置贬值比例。因此：

$$
obsol_{i,t} = \delta_i L_{i,t}
$$

$$
I_{i,t+1} = (1 - \delta_i) L_{i,t}
$$

未售出的高端 GPU 算力节点会受到多重经济因素影响：

- 新一代芯片上市；
- 市场现货价格回落；
- 资产闲置无法产出现金流；
- 机架和网络带宽固定成本持续存在。

### 1.8 收益函数 Payoff Function

项目采用如下统一收益函数对每个 Agent 进行结算：

$$
\pi_{i,t}
=
p_{i,t} x_{i,t}
+ \sum_{k \neq i} w^{tr}_{i \to k,t} m_{i \to k,t}
- \sum_{j \neq i} w^{tr}_{j \to i,t} m_{j \to i,t}
- c_i q_{i,t}
- \frac{\gamma_i}{2} q_{i,t}^2
- h_i I_{i,t+1}
- \psi_i obsol_{i,t}
- \lambda_i short_{i,t}
- \chi_i |p_{i,t} - p_{i,t-1}|
$$

其中：

- $p_{i,t}x_{i,t}$：对客户完成交付后的销售收入；
- $\sum w^{tr}m$：向同行转租或拆借节点获得的收入；
- $-\sum w^{tr}m$：为满足自家客户而向同行采购临时节点的成本；
- $c_i q_{i,t}$：线性采购或锁仓成本；
- $\frac{\gamma_i}{2}q_{i,t}^2$：凸性扩容成本，表示大规模吞吐硬件时资金和资源协调成本上升；
- $h_i I_{i,t+1}$：IDC 能耗与资金占用成本；
- $\psi_i obsol_{i,t}$：技术折旧与闲置惩罚；
- $\lambda_i short_{i,t}$：SLA 违约金与客户流失惩罚；
- $\chi_i |p_{i,t} - p_{i,t-1}|$：高频调价摩擦成本。

同行拆借价格定义为：

$$
w^{tr}_{j \to i,t} = p_{j,t}(1 + \kappa_{tr})
$$

该设定体现紧急交付情境下的二级算力溢价。

## 2. 时间序列与市场环境设计 Time Series & Market

### 2.1 市场总需求 $D_t$ 的经济含义

本场景中的总需求 $D_t$ 表示某一交易期内市场对高端 AI 算力节点的总需求，来源包括：

- 基础模型训练任务；
- 短期大规模实验；
- 推理扩容；
- 新模型发布前后的临时抢算力；
- 资本市场热点驱动的投机性订单。

高端 GPU 需求具有以下典型特征：

- 中期上升趋势，例如 AI 投资扩张；
- 周期性波动，例如训练任务排期和发布节点；
- 突发冲击，例如新品发布、出口限制、供给链扰动和资本市场情绪变化。

### 2.2 总需求生成公式

总需求由趋势项、周期项和随机项组成。

#### 趋势项

$$
B_t = \mu_0 + gt + \Delta \mathbf{1}(t \ge t_b)
$$

其中：

- $\mu_0$：初始基础需求
- $g$：长期增长趋势
- $\Delta$：结构性冲击幅度
- $t_b$：冲击开始时点

#### 周期项

$$
S_t
=
A_7 \sin \left(\frac{2\pi t}{7}\right)
+ A_{30} \sin \left(\frac{2\pi t}{30} + \phi\right)
$$

解释：

- 7 轮周期：模拟周度训练任务波动或结算节律；
- 30 轮周期：模拟中周期资金与项目排期波动。

#### 随机项 AR(1)

$$
\varepsilon_t = \rho \varepsilon_{t-1} + \sigma z_t,\qquad z_t \sim \mathcal{N}(0,1)
$$

#### 总需求

$$
D_t = \max \Big(D_{\min},\ \text{round}(B_t + S_t + \varepsilon_t)\Big)
$$

### 2.3 默认时间序列参数

$$
\mu_0 = 180,\quad g = 0.6,\quad A_7 = 18,\quad A_{30} = 10,\quad \phi = 0.3
$$

$$
\rho = 0.45,\quad \sigma = 7,\quad t_b = 35,\quad \Delta = -20,\quad D_{\min} = 50
$$

这表示：

- 高端 GPU 市场总体需求仍在扩张；
- 短期存在明显排期波动；
- 在第 35 轮可能遭遇一次宏观负冲击，例如：
- 新一代芯片预告导致旧代现货需求下降；
- 政策或出口限制扰动；
- 大客户预算阶段性收缩。

### 2.4 需求预测任务定义

每个 Agent 在轮次 $t$ 必须输出一个一步预测值：

$$
\hat D_{i,t}
$$

环境将记录其预测误差：

$$
err^{pred}_{i,t} = |\hat D_{i,t} - D_t|
$$

或平方误差：

$$
se^{pred}_{i,t} = (\hat D_{i,t} - D_t)^2
$$

预测能力会直接影响：

- 备货 $q_{i,t}$ 是否过高或过低；
- 定价 $p_{i,t}$ 是否与未来市场冷热匹配；
- 最终库存折旧与 SLA 违约水平。

## 3. 拓展机制的融入细节

### 3.1 有限理性限制 Bounded Rationality

本项目不允许 Agent 拥有全量市场历史或完全理性动态规划能力。每个 Agent 在第 $t$ 轮只能访问有限窗口历史。

#### 观测窗口

设窗口长度：

$$
W = 5
$$

则 Agent $i$ 在第 $t$ 轮的观测集定义为：

$$
\mathcal O_{i,t}
=
\Big\{
\tilde D_{\tau},\ p_{1:3,\tau},\ R_{1:3,\tau}
\Big\}_{\tau=t-W}^{t-1}
\cup
\Big\{
I_{i,t},\ \pi_{i,t-1},\ short_{i,t-1}
\Big\}
$$

其中：

- 可看到过去 5 轮公开需求报告；
- 可看到过去 5 轮所有竞争对手报价；
- 可看到过去 5 轮公开声誉；
- 只能看到自己的真实库存、上轮收益、上轮缺货。

#### 隐藏信息

以下信息不可见：

- 对手真实成本 $(c_j,\gamma_j)$
- 对手轮初库存 $(I_{j,t})$
- 未来真实需求 $(D_t)$
- AR 噪声 $(\varepsilon_t)$
- 结构突变是否即将到来

### 3.2 信息不对称与观测噪声

Agent 观测到的需求并不完全准确，而是带噪声的市场信号：

$$
\tilde D_t = \max(0,\ D_t + \nu_t),\qquad \nu_t \sim \mathcal N(0,\sigma_{obs}^2)
$$

默认：

$$
\sigma_{obs} = 5
$$

这模拟现实中的：

- 市场传闻；
- 第三方算力报告偏差；
- 代理商私下信息不完整；
- 订单需求确认延迟。

### 3.3 短视规划限制

每个 Agent 仅进行一步或短视规划，本方案固定为：

$$
H = 1
$$

即 Agent 在每轮只对本轮做最优响应，不做多期动态规划。这样可确保：

- LLM 推理负担受控；
- 决策行为更接近现实中的滚动经营；
- 模型实验可复现。

### 3.4 声誉机制 Reputation

设声誉 $R_{i,t} \in [0,1]$。

#### 履约率

$$
service_{i,t}
=
\frac{x_{i,t}}{\max(1,y_{i,t})}
$$

#### 恶意倾销判定

定义边际扩容量近似：

$$
mc_{i,t} = c_i + \gamma_i q_{i,t}
$$

若满足：

$$
dump_{i,t}
=
\mathbf{1}
\left(
p_{i,t} < mc_{i,t}
\land
p_{i,t} < 0.85 \cdot median_{j \neq i}(p_{j,t})
\right)
$$

则认定为恶意倾销。对应含义是：低于自身边际成本并明显低于同行价格中位数，属于以破坏性价格战抢夺算力客户。

#### 严重违约判定

$$
default_{i,t}
=
\mathbf{1}
\left(
\frac{short_{i,t}}{\max(1,y_{i,t})} > 0.1
\right)
$$

即若最终无法交付超过 10% 的已分配需求，则认定为严重 SLA 违约。

#### 协作贡献度

$$
help_{i,t}
=
\frac{\sum_{k \neq i} m_{i \to k,t}}
{\max(1,\ surplus^{(0)}_{i,t})}
$$

若 Agent 在自己有富余节点时主动支援同行，则可获得正向声誉增益。

#### 声誉得分

$$
score_{i,t}
=
clip\left(
service_{i,t}
+ 0.2 \cdot help_{i,t}
- 0.4 \cdot dump_{i,t}
- 0.6 \cdot default_{i,t},
\ 0,\ 1
\right)
$$

#### 声誉更新

$$
R_{i,t+1}
=
clip\left(
(1 - \eta_R) R_{i,t} + \eta_R score_{i,t},
\ 0,\ 1
\right)
$$

其中：

$$
\eta_R = 0.25
$$

### 3.5 声誉对市场和合作的影响

声誉通过两条路径影响市场演化。

#### 路径一：影响订单份额

$$
a_{i,t} = b_i + \beta_R R_{i,t} - \beta_p p_{i,t}
$$

声誉越低，单位价格相同时获得的客户订单越少。

#### 路径二：影响同行拆借成功率

$$
P^{\text{coop}}_{j \to i,t}
=
\sigma \left(
\alpha_0 + \alpha_1 R_{i,t}
- \alpha_2 \frac{y_{j,t}}{A_{j,t} + 1}
- \alpha_3 dump_{i,t-1}
\right)
$$

声誉越差，同行越不愿在关键时刻借出节点帮助其交付大客户。于是，早期价格战会通过后续拆借能力下降反噬自身利润。

## 4. 仿真流程设计：至少 10 轮交互

本项目采用固定单轮生命周期，所有环境状态与经济结算由统一环境对象完成。最低运行轮数为 10，正式实验建议 60。

### 单轮交互流程

#### Step 1：环境生成当轮真实市场需求

依据时间序列公式生成：

- $B_t$
- $S_t$
- $\varepsilon_t$
- $D_t$

#### Step 2：生成带噪声的市场观察信息

环境将真实需求转化为 Agent 可见的公共市场信号 $\tilde D_t$，并拼接过去 $W=5$ 轮历史形成观测窗口。

#### Step 3：Agent 接收有限历史信息

每个 Agent 仅获得：

- 最近 5 轮需求信号；
- 最近 5 轮公开价格；
- 最近 5 轮公开声誉；
- 自己当前库存和上轮利润。

#### Step 4：Agent 内部推理与需求预测

每个 Agent 基于 $\mathcal O_{i,t}$ 输出：

$$
\hat D_{i,t}
$$

作为本轮对市场需求的预测。

#### Step 5：Agent 输出行动

每个 Agent 必须输出两项动作：

- 价格 $p_{i,t}$
- 新增节点量 $q_{i,t}$

为保证复现性，输出使用离散动作索引或固定 JSON 结构。

#### Step 6：环境校验动作合法性

环境验证：

- 报价是否在合法网格中；
- 节点量是否在合法范围内；
- 若非法，则映射到最近合法动作，或回退至基线策略。

#### Step 7：环境进行订单分配

根据 $D_t$、$p_{i,t}$、$R_{i,t}$ 和 $b_i$ 计算：

- 各家吸引力 $a_{i,t}$
- 订单份额 $s_{i,t}$
- 分配需求 $y_{i,t}$

#### Step 8：识别缺口与启动同行拆借

计算：

- 初始缺货 $short^{(0)}_{i,t}$
- 初始富余 $surplus^{(0)}_{i,t}$

对存在缺口的 Agent 发起算力拆借请求，其他 Agent 根据合作概率决定是否提供节点。

#### Step 9：环境进行最终交付与经济结算

计算：

- 最终销量 $x_{i,t}$
- 最终违约量 $short_{i,t}$
- 轮末闲置节点 $L_{i,t}$
- 技术折旧 $obsol_{i,t}$
- 下轮库存 $I_{i,t+1}$
- 利润 $\pi_{i,t}$

#### Step 10：环境更新声誉并记录日志

依据履约率、违约事件、倾销事件与协作行为更新 $R_{i,t+1}$，并将所有变量写入实验结果文件。

## 6. 实现约束与可复现性要求

为保证本项目具备高度代码可复现性，实施时遵循以下约束。

### 6.1 动作输出结构化

LLM Agent 输出必须是固定 JSON，例如：

```json
{
  "forecast_demand": 196,
  "price_idx": 11,
  "qty_idx": 7
}
```

环境负责将索引映射为实际价格和节点量。

### 6.2 决策温度固定

所有 LLM 决策调用统一设定：

- `temperature = 0`
- 固定 system prompt
- 固定 observation schema

### 6.3 环境统一结算

所有收益、库存、拆借、违约和声誉更新必须由 `Environment.step()` 统一执行，Agent 不允许自行修改状态变量。

### 6.4 非法输出回退机制

若输出解析失败或越界，则自动：

1. 映射到最近合法动作；
2. 若仍失败，退回到上一轮动作；
3. 若第 1 轮无上一轮动作，则使用预设 baseline policy。

## 7. 结论

本方案将 PJ-AG4 具体化为一个具有现实感和研究张力的高端 GPU 算力现货与供应链博弈系统。在该系统中：

- 库存被建模为高价值算力资产；
- 腐损被定义为技术折旧与资产闲置贬值；
- 缺货对应 SLA 违约与客户流失；
- 同行调货扩展为算力联邦和集群节点拆借；
- 声誉会直接影响订单份额和紧急拆借能力。

因此，该项目既适合作为课程大作业，也具备较强的科研表达能力：它能够系统性研究 LLM Agent 在有限理性、动态需求、声誉约束和重复竞争环境中的策略形成与演化规律。
